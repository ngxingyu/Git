Revising your repository

Revisiting past commits

This command allows you check out old versions of files, view previous states of the project, or check out various branches of the project with the commit's ID.
The ID of the commit can be obtained using the git log --oneline command.
> git log --oneline
For now, we will look at the first two uses of the checkout command.

Individual files

Firstly, suppose you want to check out an old version of a file. Use git checkout <commit ID> <filename>. In this case, I want to view the state of my file text.txt at a certain commit: 1kjh23k.
> git checkout 1kjh23k text.txt
Now, when you view the file text.txt, it will be in the state of the 1kjh23k commit.
If you want to keep this version of the file, you can use the git commit command.
Otherwise, if you do not want to keep that version of the file, just use git checkout HEAD <filename>. In the case of text.txt,
> git checkout HEAD text.txt

Entire commits

If you want to revisit your project at a previous commit: 1kjh23k, you can use the command git checkout <commit ID>.
This reverts the entire working directory to its 1kjh23k commit state, allowing you to view, edit, or even run files without losing the current project state.
To return to the current state of the project:
> git checkout master

Reverting your committed changes

If you need to undo a commit, Git would append a new commit with the reverted changes specified by the commit. This is done with the help of git revert
To revert the changes made by the last active commit, use the following command:
> git revert HEAD
In addition, you can revert the changes of any commit by specifying how many commits away it is from the latest commit. For instance, to revert to the 2nd last commit made:
> git revert HEAD~1

Resetting your committed changes

You can easily unstage files, reset the staging area, remove all uncomitted changes, and even remove committed files with the git reset command.

Unstage single files

To unstage a specific files, you can simply use the git reset <filename> command. This would leave every other part of the working directory unchanged.
> git reset text.txt

Unstage all files

To unstage all staged files, just use the command git reset. This would also leave every other part of the working directory unchanged.
> git reset

Remove all uncommitted changes

To unstage all staged files and remove all uncommitted changes, add the ‘--hard’ flag after the git reset command. This would remove all uncommitted changes permanently.
> git reset --hard

“Un-commit” selected files

The command git reset <commit ID> will shift all changes made after that commit to the working directory, allowing you to re-commit all of them.
> git reset 1kjh23k

Removing committed files

The command git reset --hard <commit ID> will remove all uncommitted changes and commits made after that specified commit permanently.
> git reset --hard 1kjh23k

Removing untracked files

The command git clean will remove all untracked files.
> git clean
