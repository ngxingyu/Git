Git: An Introduction

What is Git?

Git is a free version control system.

What are Version Control Systems?

Version Control refers to a system which keeps track of different versions of certain files or programs such that they can be easily recalled later. This way, if you make any mistake to your code, or find a bug within your program, you are able to recover any older versions of the file to debug your code with ease.
There are several types of Version Control Systems:
Local Version Control Systems: This is the most basic form of version control, where different versions of files are saved locally.
Centralized Version Control Systems: This is a version control system which saves different versions of files to a server, allowing different users to access the files simultaneously. However, with only a single server being utilized, if anything happens to the server, all your files would be lost.
Distributed Version Control Systems: Hence, distributed version control systems like git are most preferred. Different clients now copy the entire repository to their computers. In the case where any server dies, the full repository can be restored from any client's computer. Thus, multiple backups are present.
While most VCS store information as changes made to individual files, Git records the changes to the entire project as a whole, linking unchanged files to a previous identical file it has already stored.
