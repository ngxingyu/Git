Basics

Initialize your Git Repository

To start tracking a project in Git, type git init within the project's directory. This initializes a repository which would contain all your files.
> git init

Start Tracking new files

Then, to start tracking a new file in your repository, use git add. Suppose you want to start tracking a file named text.txt, you can enter 
> git add text.txt
Now, your file has been tracked and staged
You are also able to use the asterik sign (*) to add multiple files of the same file type. For instance, to add all text files (.txt), you can enter 
> git add *.txt

Check your status

You can use git status to view the state of your project at any point. 
> git add text.txt
This would allow you to determine the state of every file within the repository.
This is how your status will look after you initialize your repository.
> git status
On branch master
nothing to commit, working directory clean
And this is how your status will look like if you add a new file (text.txt) to your repository. 
> git status
On branch master
Untracked files: 
  (Use “git add <file>...” to include in what will be committed)

  text.txt

nothing added to commit but untracked files present (use “git add” to track)
