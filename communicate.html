Communicating with other repositories

Adding remote repositories

To collaborate with others on your project, you make use of remote repositories hosted on networks.
To start, you have to add a remote repository. You can do so using the following command: git remote add <name> <url>
> git remote add abc https://www.website.com/tomandjerry/repo.git
Then, now that you have added a remote repository, you can view the remote repositories you have connected with by entering git remote.
> git remote
abc
You can add a '-v' flag to the above command if you also want to view the URLs of the remote repositories.
> git remote -v
abc https://www.website.com/tomandjerry/repo.git

If you want to disconnect any remote repositories, use the command git remote rm <name>.
> git remote rm abc
To rename any remote repositories, use git remote rename <old name> <new name>.
> git remote rename abc cba

Accessing remote repositories

Now that you have added remote repositories, you will want to check it out. You can use git fetch <remote name> to obtain all the commits and files from the repository.
> git fetch cba
To fetch data from a specific branch in the repository, use git fetch <remote name> <branch name>.
> git fetch cba tom
Now, you are able to view the files you fetched on a remote branch. You can review commits before adding them to your local repository. To merge the remote branch with your current branch, use git merge.
Alternatively, you can use git pull <remote name> <branch name> to fetch and merge files and commits from remote repositories
> git pull cba tom

The opposite of git fetch is git push. As you can guess, this command allows you to transfer commits from your local repository to other remote repositories.
To push data to a specific branch in the repository, use git push <remote name> <branch name>.
> git push xyz mybranch
To push all your branches to the remote repository, use git push <remote name> --all.
> git push xyz --all
