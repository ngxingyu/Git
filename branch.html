Branches

What are branches?

Branches are lines of development which are independent of the main line.
These branches provide a platform where you can modifications or commits without affecting the original line of development.
When you use the git init command, the master branch will be created automatically.
To create a new branch for you to work on, use git branch <branch name>. If I want to have a new branch called 'test'
> git branch test
Use the command git branch to view all the branches present in your repository.
Then, to start work on the new branch, you have to checkout the branch.
> git branch test

> git branch
master
test

> git checkout test

Rather than using two steps to create and visit a new branch, you can make use of a single command: git checkout -b <branch name> to create and visit a new branch.
> git checkout -b test

Merging your branches

After you have made your modifications in the new branch and feel happy with it, you would want to save you changes.
To do so, you can switch back to your current branch, and merge the changes from the new modified branch to the current branch. Just use git merge <new branch name>
> git checkout master
> git merge test
This would make the master branch identical to the test branch.

Now that you have successfully made your changes to the master branch, the test branch is unneccessary, and you can remove it with git branch -d <branch name>. 
> git branch -d test

Integrating changes between branches

Another way you can integrate the changes you made from the test branch to your master branch is through git rebase.
Since the test branch is essentially a copy of the master branch, you can take the changes you make to the test branch, and apply it to the master branch.
You do this by checking out the new branch, then rebase any changes made to the new branch onto the current branch. Then, return to the current branch, and merge both branches.
> git checkout test
> git rebase master
> git checkout master
> git merge test
Now, you have a history in which all changes you actually made to the test branch is made to the master branch, giving you a linear history, instead of a branched one.

Stashing your work

Suppose you are half-way through working on a section of your project, and you need to visit another branch, but do not want to commit your unfinished changes.
You can make use of git stash to save your current working state, allowing you to do whatever you want. Just enter the following command
> git stash
This would save your changes. To view all the stashes you have, use git stash list, and it will return a list of stashes.
> git stash apply
stash@{0}: WIP on master: ...
stash@{1}: WIP on master: ...

To reapply the last stash, just use git stash apply. Else, for any other older stashes, use git stash apply stash@{1} etc...
