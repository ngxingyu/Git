Adding to your repository

git add

The command git add has multiple purposes.
As mentioned earlier, git add can be used to start tracking new files. In addition to that, git add has other purposes, including staging files.
Suppose you modify the text.txt file that is already tracked, git status will return
> git status
On branch master
Changes to be committed:
  (use “git add <file>...” to unstage)

    new file:  text.txt
Use the command git add to stage the latest version of the file in question (in this case, text.txt)
> git add text.txt

Committing your changes

Now that you have staged your modified files, git commit will allow you to commit these changes.
The default git commit command will launch a text editor in which you can input your custom commit message.
Alternatively, you can add your commit message inline, with the following command
The commit message does not have any formatting constraints, but it should provide a concise description of the changes made.
> git commit -m "Add your commit message here"

[master 1kjh23k] Add your commit message here
1 file changed, 1 insertions(+)
create mode 100644 text.txt

> git commit -a -m "message"
You are also able to commit any changes present by adding ‘-a’. This achieves the same effect as staging and committing all modified files.

Reviewing your changes

Now, your file has been committed. You can check the status of your files with git status as mentioned earlier. To view the history of all the commits you made over time, make use of git log. This will provide you with the entire list of commits you made, along with its commit message, and timestamp.
Here is an example of the git log command.
> git log
commit 1kjh23k598sdg73nth359nxdhfne8e8g7234h223
Author: Xing Yu Ng <ng.xingyu@dhs.sg>
Date: Fri Dec 18 15:06:14 2015 +0800

Add your commit message here

You can easily customize your log output using the following commands.

Using git log -n *integer*

With this command, you can specify the number of commits to display.
> git log -n 2
For instance, the above code displays the latest two commits.

Using git log --oneline

This command condenses each commit to a single line, just displaying the checksum and commit message.
> git log --oneline

Using git log -p

This command displays the full diff of every commit, providing all details of the project history.
> git log -p

Using git log <file>

This command displays the commit for the specified file only.
> git log text.txt
